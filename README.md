## 📖 Skill Descriptions

### 1. **Design-to-Code Skill**
Transforms screen or component descriptions into clean handoffs for developers. Produces:
- Clear functional and UI specifications
- HTML/CSS or Tailwind code snippets
- Dev checklist for design fidelity verification

**When to use:** After UX/UI specs are finalized and a dev handoff is needed.

---

### 2. **Nielsen Heuristic Evaluator** (PRO Version 2026)
Conducts complete UX evaluations using Jakob Nielsen's 10 heuristics. Features:
- 5-step systematic evaluation process
- Scores from 1-5 with concrete evidence
- Weighted scoring with SUS estimation
- 90-day actionable roadmap for fixes
- Integration with Figma, analytics, and accessibility standards

**When to use:** For interface audits, prototype reviews, or comprehensive UX analysis.

---

### 3. **PRD UX Skill**
Frames product problems before moving to design. Clarifies:
- Target users and use cases
- Business objectives and success metrics
- Constraints and critical journeys
- Structured PRD with personas, user stories, and UX principles

**When to use:** Before any UX or UI design phase when PRD is missing or vague.

---

### 4. **SEO UX Skill**
Optimizes interfaces with a dual SEO + UX approach:
- Hierarchical H1–H2–H3 structure aligned with search intent
- SEO metadata (titles, descriptions, OG tags)
- Content readability and user clarity enhancements
- Microcopy optimization for CTAs and guidance

**When to use:** For marketing pages, blogs, documentation, and product landing pages.

---

### 5. **UI Design System Skill**
Transforms wireframes into detailed UI screens using design systems:
- Component and token specifications
- Visual variant recommendations
- State handling (normal, hover, focus, disabled, error)
- Responsive design considerations (mobile-first)

**When to use:** When wireframes exist and consistent, tokenized UI design is needed.

---

### 6. **UX Analytics & Experiments Skill**
Bridges data and design with:
- KPI definition and tracking plans
- Funnel analysis and drop-off identification
- A/B test ideas with hypothesis and risk assessment
- Data-driven design iteration recommendations

**When to use:** For live or pre-launch products needing measurement and optimization.

---

### 7. **UX Content & Microcopy Skill**
Improves all interface texts:
- Button labels, placeholders, error messages
- Onboarding and empty state copy
- Tone of voice consistency across screens
- Before/after comparison tables with variants

**When to use:** When interface texts need clarity, reassurance, or brand alignment.

---

### 8. **UX Designer Skill**
Senior UX design deliverables from PRD or feature descriptions:
- Clear user flows (numbered steps per scenario)
- Text wireframes with sections and hierarchy
- Key states (empty, error, loading, success)
- WCAG 2.1 AA accessibility recommendations

**When to use:** After PRD approval, before moving to UI or visual design.

---

### 9. **Visual Quality Assurance**
Design-to-implementation comparison tool:
- Side-by-side Figma vs. live demo/screenshot analysis
- Classification by theme (content, design system, layout, typography)
- Priority matrix (P1–P4 critical to low)
- Saffron Design System compliance checking
- Structured VQA report with fix recommendations

**When to use:** Before shipping a feature to verify implementation fidelity against design.
